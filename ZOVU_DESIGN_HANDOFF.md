# ZOVU — Design Handoff для Claude Code

> Читается **вместе с `ZOVU_PROMPT.md`**. Этот файл объясняет, как обращаться с экспортом из Claude Design (папка `design/`). При конфликте: продуктовые правила и скоуп — из `ZOVU_PROMPT.md`; визуальные значения (цвета, отступы, радиусы, тексты экранов) — из `design/standalone.html`.

---

## 0. Главное правило (не нарушать)

Экспорт технически собран на движке дизайн-системы **«Polaris» (Shopify)** — папка `design/_ds/`. **Это НЕ визуальный стиль Zovu.** Реальные экраны в `design/standalone.html` собраны в iOS-минимализме по мокапам заказчика: синий акцент, белые карточки, тонкие бордеры. Polaris-эстетику (near-black «brand», плотная админ-типографика Inter 13px, Shopify-иконки, бевел-кнопки) **не переносить**.

- **Источник истины по стилю** — `design/standalone.html` (готовый React-прототип) и мокапы `design/mockups/zovu1–4.png`.
- **`design/_ds/`** — только техсправочник: имена токенов, шкала отступов (4px база), тайминги моушна. Значения брать те, что реально применены в standalone (см. ниже), а не дефолты Polaris.
- Если сомневаешься между Polaris-значением и standalone-значением — **всегда standalone**.

## 1. Что лежит в `design/`

| Путь | Что это | Как использовать |
|---|---|---|
| `standalone.html` | Готовый React-прототип всех экранов (UMD React + Babel, инлайн). **Главный источник стиля и разметки.** | Открой, найди компоненты экранов, срисуй цвета/отступы/иерархию/тексты в Flutter. |

| `screenshots/*.png` | PNG-превью канвасов и потоков. | Быстрый визуальный обзор без парсинга HTML. |

## 2. Реальные токены (извлечены из `standalone.html` — это канон палитры)

Перенеси в `lib/core/theme/tokens.dart`. Значения ниже — то, что фактически применено в прототипе; они совпадают по духу с разделом 4 `ZOVU_PROMPT.md`, но **используй именно эти хексы** для 1:1 совпадения с тем, что видел заказчик:

```
primary        #4C6FFF   // основной акцент (кнопки, ссылки, активные состояния)
primarySoft    #EEF1FF   // фон выбранных карточек/чипов, подложка акцента
ink            #141824   // основной текст (заголовки)
inkStrong      #16181D   // максимально тёмный текст
inkSecondary   #6B7280   // вторичный текст
inkMuted       #9AA0AD / #787E8C   // подписи, плейсхолдеры
border         #E4E6EC / #ECEDF1   // хайрлайны
surface        #F5F6F8 / #F7F7F8   // подложки секций
bg             #FFFFFF
success        #16A34A   +  successSoft #E7F6EC
warning        #E8981F / #B45309   +  warningSoft #FBEFDD
divider        #F0F1F4
```

Статус-пиллы (лежат готовыми в канвасе «Состояния» / «Компоненты»): Новый → primary на primarySoft; Ожидание ответа → warning; Принят/Выполнен → success; Не выбран/Отменён → inkMuted на surface; В работе → primary; На рассмотрении → warning. **Сверься с секцией «СТАТУС-ПИЛЛЫ» в standalone.**

Моушн (из `_ds/tokens/motion.css`, годится как есть): длительности 100–200 мс, ease `cubic-bezier(0.25,0.1,0.25,1)`, ease-out `cubic-bezier(0.19,0.91,0.38,1)`. Это база; поверх — Tinder/Duolingo-физика и haptics из раздела 4 `ZOVU_PROMPT.md` (их в экспорте нет, добавляешь ты).

Типографика: в проде — **SF Pro (iOS) / Inter (Android)** по `ZOVU_PROMPT.md`, НЕ полярисовский 13px-admin-scale. Из канваса бери только иерархию (что заголовок, что цена, что подпись), а размеры — по разделу 4.1.

## 3. Покрытие экранов из Design → маппинг на S-XX

Экспорт покрывает почти всю карту `ZOVU_PROMPT.md §5`. Сопоставь и реализуй по S-нумерации из промпта:

- **Онбординг** (канвас «Онбординг», screenshots `onb-*`): Welcome, Phone, OTP, Role, Анкета специалиста, Загрузка диплома, Верификация (селфи+документ), «Диплом на проверке» / «Проверяем данные», Success → **S-01…S-08**.
- **Заказчик** (канвас «Заказчик»): Создание заказа, Фильтры, Карта, Список откликов, Деталь отклика, Активный заказ, Завершение, Оценка → **S-20…S-27**.
- **Специалист** (канвас «Специалист»): Карта, Лента (Новые/Рядом/Все), Карточка заказа, Отклик (с комиссией «Вы получите»), Статусы откликов, Баланс, Пополнение, Блокировка «Пополните баланс», Профиль → **S-10…S-18**. Tinder-колода в экспорте НЕ отрисована — добавляешь по §4.3 промпта поверх этих экранов.
- **Общие** (канвас «Общие»): Чат, Поддержка (с категориями Заказ/Оплата/Жалоба/Верификация), Уведомления, Отзывы, Роль, Настройки (язык RU/KZ) → **S-30…S-35**.
- **Состояния** (канвас «Состояния»): empty/loading/error, статус-пиллы, блокировки, «Низкий баланс». Реализуй эти состояния для каждого списочного экрана.
- **Компоненты** (канвас «Компоненты»): Кнопки, Инпуты, Бейджи, Пиллы, карточки — это твой чеклист для UI-kit в **майлстоуне M1** и экрана `/dev/uikit`.

Сверь свою реализацию с канвасом «Карта экранов» — там граф переходов; переноси диплинки и навигацию в GoRouter по нему.

## 4. Что делать со стилем прототипа технически

1. Прототип — **React inline (UMD+Babel)**. Не конвертируй его в прод-код и не тащи React в мобильное приложение. Он — визуальная спецификация; целевой UI пишется на **Flutter** (§2 промпта).
2. Извлеки из `standalone.html`: палитру (см. §2 выше), радиусы, отступы (кратны 4), структуру каждого экрана (какие блоки, в каком порядке), точные русские тексты (сверь с Приложением A промпта — при расхождении бери standalone, он новее).
3. Собери из этого `tokens.dart` + UI-kit (M1), затем экраны (M3–M7). Каждый экран Flutter должен визуально совпадать со своим аналогом в standalone.
4. `_ds/_ds_bundle.js` и Polaris-компоненты в прод НЕ идут. Если нужен паттерн (например, как сделан бейдж) — подсмотри, реализуй нативно во Flutter.
5. Иконки: Polaris-иконки НЕ используй (они Shopify-стилистики). Бери системные Cupertino/Material или единый линейный набор, совпадающий с мокапами (см. иконки таббара, категорий, статусов на zovu1–4.png).

## 5. Приоритет источников (шпаргалка при любом конфликте)

1. **Скоуп, домен, бизнес-правила, B2B-исключения** → `ZOVU_PROMPT.md` (жёстко).
2. **Точные цвета / отступы / радиусы / тексты экранов** → `design/standalone.html`.
3. **Тон и общий вид, если standalone неоднозначен** → `design/mockups/zovu*.png`.
4. **Граф переходов между экранами** → канвас «Карта экранов».
5. **Имена токенов и шкалы (структура, не значения)** → `design/_ds/tokens/*.css`.
6. Polaris-эстетика (near-black, admin-density, Shopify-иконки, бевел) → **игнорировать всегда**.

Любое отступление — ADR в `docs/09-decisions.md`, без остановки на вопросы.
