## 1. Одной строкой

C2C-маркетплейс услуг (Казахстан, модель Naimi.kz): заказчик публикует заказ с
ценой и гео → специалисты рядом видят его в ленте/на карте и откликаются. Две роли
на одном аккаунте. **Все 9 майлстоунов M0–M8 закрыты**, Definition of Done выполнен.

- **Клиент:** React PWA (`apps/web`) — Vite + React 19 + TS, React Router,
  TanStack Query, Zustand, i18next (ru+kk), socket.io, Leaflet. Стек — по
  [ADR-008](09-decisions.md) (осознанная замена Flutter, не «пока не сделали»).
- **Бэкенд:** NestJS 10 + Prisma + PostgreSQL (`apps/api`) — **реальный, не моки**.
- **Админка:** `apps/admin` — Vite + React, вход по токену.
- **Тесты:** 33 jest-теста бизнес-правил зелёные. E2E happy-path проходит.

## 2. Как запустить (кратко)

```bash
# PG17 на :5434 (локальный кластер, без Docker — ADR-005)
cd apps/api && node dist/main.js          # API :3000 (собранный)
cd apps/web && npm run dev                 # PWA :5173 (proxy /v1 и /chat → :3000)
cd apps/admin && npm run dev               # админка :5174
# OTP в dev всегда 1111 · dev-быстрый вход обеих ролей на Welcome
```

Демо-данные: `cd apps/api && npm run seed:demo` (6 специалистов Алматы + заказы +
принятый заказ с чатом + завершённый заказ для демо отзыва).

## 3. Роли и навигация

Один аккаунт — две роли, переключение без перелогина. Два таббар-шелла:

| Шелл заказчика (`/client`) | Шелл специалиста (`/sp`) |
|---|---|
| Мои заказы · Отклики · Чаты · Профиль | Карта · Заказы · Отклики · Профиль |

## 5. Сквозные флоу, которые РАБОТАЮТ end-to-end

- **Happy-path заказчика:** онбординг → создать заказ → получить отклики → принять →
  чат → завершить → оценить. ✅ Без правки кода между шагами.
- **Happy-path специалиста:** онбординг → верификация (авто-approve) → лента → отклик →
  принятие → чат → завершить. ✅
- **Денежный цикл:** отклик при пустом балансе → 403 блок → экран блокировки →
  пополнение → разблокировка → комиссия при принятии. ✅
- **Realtime-чат** по WebSocket (две вкладки), галочки прочтения. ✅
- **Карты** Leaflet+OSM с ценовыми пинами (обе роли). ✅
- **Sprint 3A:** safety-слой, условный отзыв, маскирование контактов в откликах. ✅
- **RU↔KZ** переключение языка на лету. ✅ · **Админка** (5 очередей модерации). ✅

## 6. Что умеет бэкенд (NestJS + Prisma)

- **auth** — OTP + JWT с refresh-ротацией, guards.
- **users** — профиль, роли, анкета специалиста, верификация/диплом (multipart,
  приватный бакет).
- **orders** — создание, feed с гео-фоллбэком (`earth_distance`/haversine) + фильтры
  Ф-02…Ф-07, hide-свайп, **отмена (ЗВ-07, разные ветки до/после принятия)**.
- **bids** — отклик со структурой (цена/доступность/материалы/коммент), комиссия
  (ADR-001), **каскад «Не выбран»** при принятии + чат + списание, decline, streak,
  маскирование контактов.
- **billing** — баланс, транзакции, пополнение, cron подписки (00:00 Asia/Almaty),
  блок отклика при неактивной подписке.
- **сделка** — завершение (ЗВ-01/02/04), cron автозакрытия 24ч, отзывы (однократно,
  стоп-словарь, пересчёт рейтинга), чат Socket.IO, уведомления.
- **support** — тикеты (СП-01…10), спор (ЗВ-06 → disputed).
- **admin** — 5 очередей (верификация/дипломы/категории/жалобы/тикеты) + действия,
  block/warn/unblock, аудит-лог.
- **uploads** — фото заказа (загрузка с клиентским сжатием + публичная раздача с
  guard от path-traversal).
- **33 jest-теста** бизнес-правил зелёные.

## 7. Что осталось доделать (пробелы, НЕ переписывание)

- **G2** — кнопка отмены заказа + `ConfirmDialog` (бэкенд уже готов).
- **G3** — тред поддержки + многошаговая жалоба с выбором причины.
- **G4** — «мои активные» по статусам.
- **G5** — permission-primers гео/push/камера (локальные).
- **G6** — контрпредложение по цене (единственная новая миграция Prisma).

**Вне скоупа:** серверный web-push, платежи/эквайринг, маркеры специалистов на карте
заказчика (нужна персистентная гео специалиста).
